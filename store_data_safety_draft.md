# Google Play "데이터 보안(Data safety)" 섹션 작성 가이드

Play Console → 앱 콘텐츠 → 데이터 보안 에서 아래 순서대로 답변하시면 됩니다.

## 1. 데이터 수집/공유 여부
**"내 앱이 사용자 데이터를 수집하거나 공유하나요?"** → **예**

## 2. 데이터 유형별 체크리스트

### 개인정보 (Personal info)
- **이름 (Name)**
  - 수집함: 예 (포인트 매칭소 이용 시, 이용자가 직접 입력)
  - 공유함: 아니요
  - 선택적/필수: 선택 (매칭소를 이용하지 않으면 수집 안 됨)
  - 수집 목적: **앱 기능(App functionality)**

> 부서/직급은 Play Console 항목 중 정확히 대응되는 카테고리가 없으면 "이름"과 함께 "기타 정보(Other info)"로 묶어서 신고하셔도 됩니다.

### 앱 활동 (App activity)
- **앱 상호작용 / 기타 사용자 생성 콘텐츠 (건의사항 텍스트, 게임 진행 데이터)**
  - 수집함: 예
  - 공유함: 아니요
  - 선택적/필수: 선택
  - 수집 목적: **앱 기능**

### 기기 또는 기타 ID (Device or other IDs)
- **광고 ID (Advertising ID)** — AdMob 사용으로 인해 자동 수집
  - 수집함: 예
  - 공유함: **예** (Google AdMob에 전달되어 광고 게재/성과 측정에 사용됨)
  - 선택적/필수: 필수 (광고 표시를 위해 필요)
  - 수집 목적: **광고 또는 마케팅(Advertising or marketing)**

### 그 외 항목 (위치, 금융정보, 건강정보, 메시지, 사진/동영상, 통화기록 등)
전부 **"수집 안 함"**으로 표시하시면 됩니다. (해당 데이터를 전혀 다루지 않음)

## 3. 보안 관행 (Security practices)
- **데이터가 전송 중 암호화되나요?** → **예** (모든 통신 HTTPS)
- **사용자가 데이터 삭제를 요청할 수 있나요?** → **예**
  - 삭제 요청 방법: 개인정보처리방침(`https://www.hhimeal.com/privacy`)에 안내된 이메일로 문의

## 4. 참고
- Supabase(데이터베이스)는 앱 자체 기능을 위한 저장소일 뿐, Google이 정의하는 "제3자 공유"에는 해당하지 않습니다 (서비스 제공을 위한 위탁 처리이며, Supabase가 그 데이터를 자신의 목적으로 쓰지 않기 때문). 그래서 "공유함"에는 표시하지 않으셔도 됩니다.
- AdMob 광고 ID만 유일하게 "공유함"으로 표시하면 됩니다.
